# Contributing to MPART Augmentoolkit

## Development Workflow

1. Create a feature branch from main
2. Make your changes
3. Test locally with Docker before committing
4. Update relevant documentation
5. Submit changes with clear commit messages

## Code Standards

- Follow PEP 8 for Python code
- Add docstrings to functions
- Include comments for complex logic
- Update benchmarks.md with any performance data
- Keep configuration files well-commented

## Testing

Before submitting changes:
- Build Docker image successfully
- Run test_environment.py
- Verify documentation is up to date

## Questions?

Contact the project lead or open an issue for discussion.
